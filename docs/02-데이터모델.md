# 02. 데이터 모델 (PRD 8, 34, 36, 37)

Supabase(PostgreSQL) 기준. 모든 테이블 공통 컬럼: `id uuid pk`, `created_at`, `updated_at`, `created_by uuid`.

## 2.1 ERD 개요

```
profiles ─┐
          ├─< minwon >── minwon_history
people ───┤              minwon_comments
          ├─< contact_logs (연락 이력)
          ├─< event_attendees >── events ──< event_reports
          │
ai_requests (모든 업무 분류 로그)
pr_contents (보도자료/SNS/문자/웹자보)
policy_items (5분발언/도정질문/조례)
news_items ──< news_reports (아침/저녁보고)
files / notifications / audit_logs (횡단)
```

## 2.2 사용자 / 인증 (PRD 4, 35)

```sql
-- Supabase auth.users 와 1:1 매핑
create table profiles (
  id uuid primary key references auth.users(id),
  name text not null,
  role text not null check (role in
    ('의원','정책지원관','비서','홍보담당','조직담당')),
  phone text,
  is_active boolean default true,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

## 2.3 CRM — 인물·분류·연락이력 (PRD 7~10)

```sql
create table people (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  category text,              -- 인물 분류체계: 지역주민/유관기관/언론/당원/공무원 등
  org text,                   -- 소속 조직
  position text,              -- 직책
  region text,                -- 지역구/동
  phone text, email text, address text,
  tags text[] default '{}',
  importance int default 0,   -- 중요도(통계용)
  memo text,
  created_by uuid references profiles(id),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table contact_logs (        -- 연락 이력 (10)
  id uuid primary key default gen_random_uuid(),
  person_id uuid references people(id) on delete cascade,
  channel text,               -- 전화/문자/대면/이메일
  summary text not null,
  contacted_at timestamptz default now(),
  created_by uuid references profiles(id),
  created_at timestamptz default now()
);
```

## 2.4 민원 (PRD 11~14)

```sql
create table minwon (
  id uuid primary key default gen_random_uuid(),
  title text not null,
  content text,
  person_id uuid references people(id),  -- 민원인
  category text,                         -- 분야(교통/복지/환경 등)
  status text default '접수'             -- 접수→처리중→완료→보류
    check (status in ('접수','처리중','완료','보류')),
  priority text default '보통'
    check (priority in ('긴급','높음','보통','낮음')),
  assignee uuid references profiles(id),
  due_date date,
  created_by uuid references profiles(id),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table minwon_history (    -- 처리 이력 (13)
  id uuid primary key default gen_random_uuid(),
  minwon_id uuid references minwon(id) on delete cascade,
  from_status text, to_status text,
  note text,
  created_by uuid references profiles(id),
  created_at timestamptz default now()
);
```
> 민원 통계(14)는 별도 테이블 없이 `minwon` 집계 뷰/쿼리로 제공.

## 2.5 일정·행사 (PRD 15~18)

```sql
create table events (
  id uuid primary key default gen_random_uuid(),
  title text not null,
  type text,                  -- 행사/회의/의정활동/지역일정
  location text,
  start_at timestamptz not null,
  end_at timestamptz,
  description text,
  status text default '예정'
    check (status in ('예정','진행','완료','취소')),
  created_by uuid references profiles(id),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create table event_attendees (   -- 참석자 관리 (17)
  id uuid primary key default gen_random_uuid(),
  event_id uuid references events(id) on delete cascade,
  person_id uuid references people(id),
  attend_status text default '미정'
    check (attend_status in ('참석','불참','미정')),
  note text
);

create table event_reports (     -- 결과보고 (18)
  id uuid primary key default gen_random_uuid(),
  event_id uuid references events(id) on delete cascade,
  summary text, headcount int, photos text[],
  created_by uuid references profiles(id),
  created_at timestamptz default now()
);
```

## 2.11 RLS 정책 원칙 (PRD 45)

- 모든 테이블 `enable row level security`.
- 기본: 로그인 사용자만 `select`. 쓰기는 역할별 분리.

```sql
-- 예) 홍보 콘텐츠는 홍보담당/의원만 수정
create policy pr_write on pr_contents for all
using (
  exists (select 1 from profiles p
          where p.id = auth.uid()
          and p.role in ('홍보담당','의원'))
);
```

| 테이블 | 쓰기 허용 역할 |
|--------|----------------|
| people, contact_logs | 조직담당, 의원 |
| minwon* | 비서, 의원 |
| events, event_* | 비서, 의원 |
| pr_contents | 홍보담당, 의원 |
| policy_items | 정책지원관, 의원 |
| news_* | 정책지원관, 의원 |
| audit_logs | (서버 service-role 전용, 직접쓰기 차단) |

## 2.12 인덱스 / 통계

- `minwon(status, category, created_at)`, `events(start_at)`, `news_items(published_at)`, `contact_logs(person_id)` 인덱스.
- 통계(대시보드/민원통계)는 **SQL View** 또는 Supabase RPC로 집계 제공.
