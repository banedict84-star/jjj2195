# 04. 화면 설계 (PRD 6, 4, 각 업무 화면)

## 4.1 전체 레이아웃

```
┌───────────────────────────────────────────────────────┐
│  상단바: 로고 | 통합검색 | 알림🔔 | 사용자(역할)         │
├──────────┬────────────────────────────────────────────┤
│ 사이드바  │                                            │
│ ─────────│           메인 컨텐츠 영역                   │
│ 대시보드  │                                            │
│ AI비서실장│   (페이지별 카드/테이블/폼)                  │
│ CRM      │                                            │
│ 민원     │                                            │
│ 일정     │                                            │
│ 홍보     │                                            │
│ 정책     │                                            │
│ 뉴스     │                                            │
│ 관리     │                                            │
└──────────┴────────────────────────────────────────────┘
```
- 사이드바 메뉴는 **역할별 노출 제어**(4장). 예: 홍보담당은 홍보 중심, 의원은 전체.

## 4.3 핵심 화면 상세

### 대시보드 (6)
- 상단 KPI 카드: 미처리 민원 / 오늘 일정 / 신규 뉴스(부정) / 발행대기 홍보.
- **오늘의 AI 브리핑**: 아침보고(4단 포맷) 카드.
- 최근 활동 타임라인 + 빠른등록 버튼.

### AI 비서실장 (5) — 가장 중요
- 채팅형 입력창 + 결과는 **4단 카드**로 렌더:
  ```
  ┌ 핵심요약 ─────────────┐
  ┌ 상황분석 ─────────────┐
  ┌ 추천행동 ─────────────┐
  ┌ 다음 할 일 (체크리스트) ┐
  └ [관련 바로가기 링크]    ┘
  ```
- 카테고리 배지(민원/조직/일정/홍보/정책/뉴스) 표시.
- "다음 할 일" 항목 → 클릭 시 해당 업무 등록/이동.

### CRM (7~10)
- 목록: 검색·필터(분류/지역/태그), 중요도 정렬.
- 상세: 기본정보 + **연락이력 타임라인** + 빠른 연락 기록.

### 민원 (11~14)
- 칸반/리스트 토글: 접수→처리중→완료→보류 상태 이동.
- 접수 폼(민원인 = CRM 연결), 처리 이력 로그.
- 통계: 분야별/상태별/기간별 차트.

### 일정·행사 (15~18)
- 월/주 캘린더 + 리스트. 행사 상세에 참석자 관리, 종료 후 결과보고 작성.

### 홍보 스튜디오 (19~23)
- 좌측 입력(주제/톤/핵심포인트) → 우측 AI 생성 결과 미리보기.
- 탭: 보도자료 / SNS / 문자 / 웹자보. 채널별 길이·형식 표시.
- 초안 저장 → 검토 → 발행 워크플로.

### 정책 지원 (24~27)
- 유형 선택(5분발언/도정질문/조례검토) + 근거자료 첨부 → AI 초안.

### 뉴스 (28~31)
- 모니터링 보드: 감성(긍정/중립/부정) 필터, 키워드 클라우드.
- 아침/저녁 보고 생성 버튼 → 4단 포맷 보고서.

### 관리 (44~45)
- 사용자/역할 관리, 감사로그 조회, 알림·연동 설정.

## 4.4 디자인 원칙
- TailwindCSS 기반 컴포넌트, 반응형(데스크톱 우선, 모바일 대응).
- 의원실 사용자(비전문가) 고려 → 직관적 라벨/한국어 UI, 최소 클릭.
- 모든 생성형 결과는 **검토 후 저장/발행**(AI 결과 무검토 발행 금지).
